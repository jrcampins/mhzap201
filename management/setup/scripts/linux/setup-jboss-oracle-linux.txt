--------------------------------------------------------------------------------
Instalación de mhzap201 en Linux, con JBoss y Oracle
--------------------------------------------------------------------------------
01. Crear el directorio mhzap201, preferiblemente en /opt

    cd /opt
    mkdir mhzap201

    Si no creó el directorio mhzap201 en /opt, puede crear el enlace simbólico /opt/mhzap201

    ln -s ruta-completa-del-directorio-mhzap201 /opt/mhzap201

02. Descomprimir el archivo zip VnnRaammdd en /opt/mhzap201

    Esta acción crea el directorio /opt/mhzap201/VnnRaammdd

    unzip VnnRaammdd -d /opt/mhzap201

    chmod -R 0777 /opt/mhzap201/VnnRaammdd


03. Ajustar las variables de entorno en:

    /opt/mhzap201/VnnRaammdd/variables-conf.sh
    /opt/mhzap201/VnnRaammdd/variables-home.sh
    /opt/mhzap201/VnnRaammdd/variables-jboss.sh
    /opt/mhzap201/VnnRaammdd/variables-oracle.sh


04. Crear y otorgar privilegios al usuario MHZ en Oracle.

    /opt/oraclexe/app/oracle/product/11.2.0/server/bin/sqlplus

    Conectar con el usuario SYS:

        connect SYS/password as SYSDBA; 

        NOTA: el password de SYS es definido al instalar Oracle.

    Crear el usuario MHZ:
        
        create user MHZ identified by sesamo;

    Otorgar privilegios al usuario MHZ:
        
        grant connect, DBA to MHZ;


05. Configurar JBoss (seguir los pasos indicados en el archivo /opt/mhzap201/VnnRaammdd/configuracion-jboss-linux.txt).


06. Ejecutar el script de instalación

    06.01.  Para instalar por primera vez, ejecutar el script de instalación
            /opt/mhzap201/VnnRaammdd/jb-ora-install.sh.

            Este script lleva a cabo las siguientes acciones:

            06.01.01. chmod: otorga los permisos necesarios
            06.01.02. dos2unix: convierte los archivos de texto de formato DOS a
                      formato UNIX
            06.01.03. customize: crea paquetes de utilidad en la base de datos
            06.01.04. makedb: crea los objetos de la aplicacion en la base de datos
            06.01.05. rebuild: reconstruye las tablas "plus" correspondientes a
                      cada tabla "arbol"
            06.01.06. standalone-start: inicia la ejecucion del servidor de aplicaciones
            06.01.07. ear-deploy: implementa la aplicacion de empresa en el servidor
                      de aplicaciones

    06.02.  Para desinstalar posteriormente, ejecutar el script de desinstalación
            /opt/mhzap201/VnnRaammdd/jb-ora-uninstall.sh.

            Este script lleva a cabo las siguientes acciones:

            06.02.01. chmod: otorga los permisos necesarios
            06.02.02. dos2unix: convierte los archivos de texto de formato DOS a
                      formato UNIX
            06.02.03. standalone-start: inicia la ejecucion del servidor de aplicaciones
            06.02.04. ear-undeploy: anula la implementacion de la aplicacion de empresa
                      en el servidor de aplicaciones
            06.02.05. standalone-stop: detiene la ejecucion del servidor de aplicaciones
            06.02.06. dropdb: elimina los objetos de la aplicacion en la base de datos


--------------------------------------------------------------------------------
Operación de mhzap201 en Linux
--------------------------------------------------------------------------------
Una vez instalado el producto, el servidor de aplicaciones queda listo para permitir
su operación normal. Con su navegador web abra http://localhost:8080/mhzap201-web/
La sesión de trabajo comienza por la pagina de autenticación del usuario. Al 
instalar se define el usuario "admin" y su contraseña es "sesamo".
